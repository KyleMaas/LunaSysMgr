00026 /*
00027 
00028         Interface for virtual keyboards implemented inside system manager, so that we can have different kinds of keyboards (tablet, phone, pinyin?...)
00029         co-exist in the code base without risky overlap.
00030 
00031         To create a new virtual keyboard:
00032          1 - create a virtual keyboard derived from VirtualKeyboard, which implements the interfaces of VirtualKeyboard & InputMethod (it derives from both)
00033          2 - create a factory for it: Derive from VirtualKeyboardFactory, implement the "create()" method, and create a static object for that factory. Example:
00034 
00035                         class TabletKeyboardFactory : public VirtualKeyboardFactory
00036                         {
00037                         public:
00038                                 TabletKeyboardFactory() : VirtualKeyboardFactory("Tablet Keyboard")             {}
00039                                 InputMethod *   create(IMEDataInterface * dataInterface)                                { return new TabletKeyboard(dataInterface); }
00040                         };
00041 
00042                         static TabletKeyboardFactory sTabletKeyboardFactory;
00043 
00044          3 - IMEManager will have to be told to use that keyboard. How this happens remains tbd, but IMEManager now has the list of names & the means to construct the keyboard.
00045 
00046 */
